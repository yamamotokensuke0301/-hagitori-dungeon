## 1.0.1修正候補（優先順）

1. **[セーブ] セーブ初期化後の別タブ復活**（main.js:1631-1634, 7235-7240, 1068）
   初期化でリビジョンが0に戻るため、開きっぱなしの別タブが旧データを新リビジョンで書き戻し、削除したはずのセーブが復活する。多重タブ+初期化という現実的な操作で発生。
   修正案: 初期化時にリビジョンを0でなく「旧値+1」で書く（単調増加を維持）。
2. **[戦闘] 壁上の霊系に近接基本攻撃が届かない**（main.js:4051 vs 4062, 6737）
   壁抜け敵が壁マスに立って隣接攻撃してくるのに、movePlayerが壁判定を敵判定より先に行うため体当たり攻撃不能。攻撃系職業技を持たない職は一方的に殴られる。
   修正案: movePlayerで「目標マスに生存敵がいるか」を壁判定より先に評価する。
3. **[戦闘] 拘束・気絶・呪い・防御破壊が加速敵で速く切れる**（main.js:4934-4943, 6741 vs 6723）
   これらは敵のサブ行動ごとに減るが、鈍化だけは世界ターンごと。ログの「3世界ターン封じた」という約束が加速12以上の敵に対して事実上嘘になる。
   修正案: 鈍化と同様に世界ターン単位の減算へ統一。
4. **[生成] 守護者を壁内で倒すと階段が壁に埋まる**（main.js:5204、監査2系統が独立に確認）
   壁抜け守護者を壁内で撃破すると唯一の階段が壁マスに出現し、そのフロアを降りられない（帰還→再生成で復旧可能だが体験が悪い）。
   修正案: 階段設置時に該当マスをfloorへ掘削する。
5. **[生成] descend/startDungeonAtが生成前に状態を書き換える**（main.js:4531-4540, 3667-3671）
   generateFloorのthrow経路（低確率）で階層カウンタと実ダンジョンが食い違ったままセーブされる。
   修正案: 生成成功後に floor/deepestFloor/HP/商店更新を行う順序へ。
6. **[生成] 狂気の部屋の配置数が部屋の空きマスを超えうる**（dungeon-generator.js:243、B40以深のみ）
   TypeError→リトライ消費（通常は自然回復）。修正案: 宝物庫と同じ Math.min(desired, positions.length)。
7. **[セーブ] 破損セーブ復旧セッションがセーブ不能ループに入る**（main.js:530, 1068）
   SAVE_KEYだけ壊れリビジョンキーが残ると、復旧後の全セーブが「別タブ競合」扱いで拒否され続ける（脱出はセーブ初期化のみ）。migrateSave内のTypeError（enemies配列の非オブジェクト要素、main.js:779）も同じ連鎖に入る。
   修正案: defaultSaveへフォールバックした時はリビジョンキーも初期化。enemiesループ前にisPlainRecordで濾過。
8. **[セーブ] 将来のモンスター改名で進行中ダンジョンが破棄される**（main.js:585）
   1.0.0では無害。次にモンスターIDを変更するパッチで発火する地雷。修正案: 検証で弾かず未知IDの敵だけ間引く（metaレコード側631-635と同じ方針）。
9. **[軽微] 異変フロアの増援・召喚に異変バフが乗らない**（main.js:3873 vs 6884）/ deferredSpecialRoomsが死にコード（dungeon-generator.js:343）/ 花ペットが通路を最大13ターン塞ぐ（自然消滅で必ず解消）— いずれも許容範囲、余力があれば。
