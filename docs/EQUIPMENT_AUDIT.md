# 装備体系監査

`osascript -l JavaScript tools/equipment_audit.js` で、通常装備の完全同一性能と同一時期・同一装備職内の完全上位互換を監査する。

素材系統ごとの鎚耐性、装飾品の副攻撃属性、防具の重量特性、粗製系列の「加速度-1・一属性耐性+1」によって、装備数を減らさず完全重複を15%以下へ抑えた。

さらに普及等級以上へ猛攻・城塞・疾風・再生・多相・背水の6原型を付与した。各原型は大きな長所と短所を同時に持ち、単なる微差ではなく戦法の変更を要求する。

高等級の城塞・再生・多相型へ全15属性を循環する免疫調律を加えた。免疫は合算前の負耐性を完全に上書きするが、免疫装備自身が攻撃-2、加速度-3、別属性-2を負うため、弱点を塞ぐたび別の構成判断が生じる。

## 人力確認で見つかった主要問題

1. 鎚系は素材名だけ違い、攻撃属性まで打へ統一されるため最大40品が完全同一になる。
2. 首飾り・指輪・護符は同じ耐性テーマ同士で数値と装備職が一致しやすい。
3. 粗製・普及系列が基礎品と同じ十階帯に入り、同価格帯の上位品へ即座に負ける。
4. 属性の異なる素材でも、鎚化した時に属性個性が消える。
5. 固有連携を持つ装備は単純な数値比較から救済されるが、現状17品では不足している。

## 再設計順

1. 最大重複の鎚系へ、素材属性に応じた衝撃後効果を個別付与する。
2. 装飾品は耐性値だけでなく、耐性閾値を別能力へ変換する役割へ分ける。
3. 完全上位互換813品は削除せず、旧セーブ互換を保ったまま固有連携候補リストとする。
4. 一つの装備を改修するたび監査を再実行し、同一性能群と上位互換数の両方を減らす。
5. 数を隠す目的の整理は行わず、プレイヤーの選択肢が増える改修だけを採用する。
